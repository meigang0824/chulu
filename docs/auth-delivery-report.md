# 初炉小程序登录与权限交付说明

## 登录策略

本项目为微信小程序，普通用户不采用传统账号密码作为主登录方式。普通用户点击“微信一键进入”后，由云函数基于当前微信 `openid` 自动创建或读取买家账号，并生成 7 天有效 session。

店长端使用账号/手机号 + 密码登录，作为后台管理权限的二次校验。登录失败达到 5 次后锁定 15 分钟。

## 角色与跳转

| 角色 | 登录方式 | 权限 | 登录后跳转 |
| --- | --- | --- | --- |
| 普通用户 buyer | 微信身份登录 | 首页、分类、订单、地址、下单、取消订单 | 用户端首页或原目标页 |
| 店长 owner/manager/staff | 账号/手机号 + 密码 | 后台全部管理页、商品、订单、库存、配送、配置 | 店长工作台 |

## 新增集合

| 集合 | 用途 | 权限 |
| --- | --- | --- |
| `auth_accounts` | 账号、角色、密码摘要、锁定状态、微信 openid 绑定 | 前端不可直读直写，仅云函数读写 |
| `auth_sessions` | token 摘要、openid、角色、过期时间、活跃时间 | 前端不可直读直写，仅云函数读写 |

## 云函数接口

| action | 说明 |
| --- | --- |
| `authWxLogin` | 普通用户微信身份登录/自动建档 |
| `authLogin` | 店长账号密码登录 |
| `authMe` | 校验当前 token 并返回登录态 |
| `authLogout` | 注销当前 session |
| `forgotPassword` | 返回重置密码指引 |

后台管理类 action 现在统一要求携带有效店长 token；`createOrder` 与 `cancelOrder` 也要求有效登录 session。

## 验收点

- 未登录访问订单、地址、下单页面，会跳转登录页。
- 未登录或普通用户访问 `/pages/admin/*`，会跳转店长登录。
- 店长登录成功后进入后台工作台。
- 普通用户登录成功后进入用户端。
- 店长密码错误会提示剩余次数，第 5 次锁定账号 15 分钟。
- 退出登录后，后台与订单相关页面需要重新登录。
