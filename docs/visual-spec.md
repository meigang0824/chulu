# 视觉规范

本文件基于 `docs/prototypes/` 九张原型图整理，用于后续高保真还原。设计稿图片尺寸为 `941 x 1672`，开发按 750rpx 宽度换算。

## 整体风格

- 风格关键词：温暖烘焙、奶油白、浅米色、草莓红、柔和卡片、轻拟物食品图。
- 页面背景：顶部多为奶油米色渐变，内容区为 `#FFF8F0` 或更浅的 `#FFFCF8`。
- 视觉重心：食品大图、红色价格/按钮、白色圆角卡片。
- 图片风格：真实烘焙食品照片，暖光、浅景深、饱和度适中。

## 色彩

| 用途 | 色值 | 使用位置 |

| 主色 | `#FF4F66` | 主按钮、价格、选中 tab、红色状态、数量加号 |

| 主色浅 | `#FFF0F2` | 截单提示、标签底、图标圆底 |
| 橙色 | `#FF8A1F` | 配送、进行中、提示、次级按钮 |

| 成功绿 | `#28B65A` | 已完成、已备齐、下降指标 |
| 蓝色 | `#3A8CFF` | 配送中状态、物流类图标 |
| 背景 | `#FFF8F0` | 全局页面背景 |

## 字体层级

| 层级 | 建议字号 | 字重 | 场景 |
|---|---:|---:|---|
| 超大标题 | 52-64rpx | 700 | 首页 banner 主标题、工作台主标题 |
| 页面标题 | 36-42rpx | 700 | 导航标题、卡片区标题 |
| 卡片标题 | 30-36rpx | 700 | 商品名、模块标题、客户名 |
| 正文 | 26-30rpx | 400/500 | 描述、表单项、订单字段 |
| 辅助 | 22-24rpx | 400 | 时间、说明、库存、占位 |
| 价格 | 40-56rpx | 700 | 商品价、合计、实付金额 |

注意：原型标题多为深棕色粗体，价格和主行动按钮为草莓红；删除价使用灰棕色并加删除线。

## 布局与间距

- 页面左右安全边距：24rpx。
- 卡片间距：20-28rpx。
- 卡片内边距：24-32rpx。
- 顶部自定义导航区高度需适配状态栏，内容从状态栏下方开始。
- 商品详情大图高度约 560rpx，圆角集中在图片容器或下方信息卡衔接处。
- 底部操作栏高度约 120-160rpx，并适配 `env(safe-area-inset-bottom)`。
- 商家表单页为左右分栏视觉：左侧表单约 62%，右侧预览约 38%；在真实小程序窄屏实现时可保留原型比例，避免内容横向滚动。

## 圆角与阴影

- 小标签圆角：8-12rpx。
- 输入框/筛选胶囊圆角：28-999rpx。
- 常规卡片圆角：24-32rpx。
- 大横幅圆角：32-40rpx。
- 主按钮圆角：999rpx。
- 卡片阴影：轻柔暖棕阴影，建议 `0 12rpx 36rpx rgba(94, 49, 20, 0.08)`。
- 主按钮阴影：建议 `0 8rpx 24rpx rgba(255, 79, 102, 0.12)`。

## 导航

- 所有页面使用自定义导航，右侧保留微信胶囊占位。
- 买家首页和商家工作台使用品牌 logo + “初炉”组合。
- 二级页使用左返回箭头 + 居中标题 + 右胶囊。
- 页面标题颜色为 `#5A2416`，返回箭头为深棕。

## 卡片与列表

- 商品卡片：白底、食品图置顶或左侧，图片圆角 16-24rpx，角标为白底橙字或红/橙渐变。
- 订单/地址卡片：字段左对齐，状态标签右上，金额右侧突出，商品缩略图横排。
- 统计卡片：图标圆底、数值大号、辅助变化指标小号。
- 表单卡片：白底，输入项以浅分割线分隔，必填星号为红色。

## 按钮与状态

| 类型 | 样式 |
|---|---|
| 主按钮 | 红色水平渐变、白字、胶囊、较强视觉权重 |
| 次按钮 | 白底、棕色/红色描边、胶囊 |
| 橙色操作 | 橙色渐变或浅橙底，用于配送中、导航、提示 |
| 成功操作 | 绿色描边或浅绿底，用于已完成、再来一单 |
| 禁用/弱操作 | 浅灰或浅米底，文字使用辅助色 |

状态色：

- 已付款/待配送：红色或橙色。
- 配送中：蓝色。
- 已完成/已备齐：绿色。
- 制作中/缺口：红橙色强调。

## 图标

- 图标以线性图标为主，配合浅色圆底。
- 常见图标：返回、心形收藏、分享、时钟、货车、定位、客服、订单、加号、筛选、下载、导航、电话。
- 后续实现若引入图标库，优先使用统一线性风格；否则可用文字/简单 CSS 图标占位，但需保持大小和颜色一致。

## 图片资产要求

- 原型中的蛋糕、吐司、蛋挞、麻薯、店铺/面包篮图片是页面识别度关键。
- 后续页面实现优先从 `mock/` 数据读取图片字段；若当前 mock 缺图，应在 `complete_mock_data` 阶段补齐图片路径或远程占位。
- 首页 banner、工作台 banner、订单详情状态 banner 应使用大图或背景图，不建议仅用纯色块替代。

## 适配要求

- 全页面使用 rpx，不出现横向滚动。
- 底部 tabBar、支付栏、批量操作栏都需要兼容安全区。
- 长地址、手机号、商品名需要单行省略或自然换行，不能挤压金额/按钮。
- 表单页和商家列表信息密度较高，按钮和状态标签需固定宽度，避免文本变化导致布局跳动。
