落地说明

给设计师和前端对齐的具体规则
用户端首屏

必须优先展示真实商品图和今日开团。轮播文字覆盖图片时必须加左侧暗渐变，不能让文字直接压在亮图上。

店长端首屏

不再使用大面积品牌 Hero。顶部只保留“切回用户端”，其后直接进入统计和待处理任务。

弹窗规则

底部 sheet 用于选择类操作，居中弹窗用于确认、协议、备注。危险操作必须使用浅红背景按钮，并保留取消按钮。

按钮层级

每个页面只允许一个最强主按钮。游客登录、取消、恢复默认等必须使用白底浅边框样式。

图标规则

设计稿中符号仅用于占位，前端落地统一使用 AppIcon/uni-icons，不再混用 emoji 和手写符号。

图片规则

商品图、轮播图、Logo 均来自云存储。设计稿中的图片地址对应当前云端资源，前端仍以 cloud:// 文件 ID 入库。
